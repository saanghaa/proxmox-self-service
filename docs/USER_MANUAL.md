# Proxmox Horizon 사용자 매뉴얼

이 문서는 일반 사용자가 Proxmox Horizon에서 일상적으로 수행하는 작업을 설명합니다.  
관리자 기능은 별도 문서인 [관리자 매뉴얼](./ADMIN_MANUAL.md)을 참고합니다.

## 1. 사용자 경험 설계 철학

Proxmox Horizon의 사용자 화면은 다음 원칙을 기준으로 설계되었습니다.

- 단순한 요청 흐름
  - 사용자는 하이퍼바이저 구조를 몰라도 VM을 요청할 수 있어야 합니다.
- 필요한 정보만 노출
  - Proxmox 내부 구현보다 “내가 무엇을 할 수 있는지”가 먼저 보여야 합니다.
- 민감 작업은 더 강하게 보호
  - 키 다운로드, 키 교체 같은 작업은 OTP 재인증을 요구합니다.
- 운영 정책은 화면보다 우선
  - 사용자가 입력할 수 있는 값이라도 실제 배포는 관리자 승인과 정책 검증을 거칩니다.

## 2. 화면 구성

주요 사용자 화면:
- 로그인: `app/src/views/login.ejs`
- OTP 입력: `app/src/views/otp.ejs`
- OTP 등록: `app/src/views/otp-setup.ejs`
- 대시보드: `app/src/views/index.ejs`

관련 라우트:
- `GET /auth/login`
- `GET /auth/otp`
- `GET /`

## 3. 로그인과 계정 보호

### 3.1 로그인 절차

로그인 순서:
1. 이메일과 비밀번호 입력
2. 비밀번호 변경 필요 여부 확인
3. OTP 미등록 시 OTP 등록
4. OTP 등록 사용자는 OTP 코드 입력 후 로그인 완료

특징:
- OTP 등록이 필요한 계정은 일반 로그인만으로 대시보드에 진입할 수 없습니다.
- 비밀번호 변경 강제 상태라면 먼저 새 비밀번호를 설정해야 합니다.

### 3.2 OTP와 복구코드

OTP 관련 화면:
- OTP 입력: `app/src/views/otp.ejs`
- OTP 등록: `app/src/views/otp-setup.ejs`
- 복구코드 입력: `app/src/views/otp-recovery.ejs`

복구코드 특징:
- OTP 기기를 분실하거나 교체한 경우 사용할 수 있습니다.
- 사용 후 즉시 폐기됩니다.

## 4. 언어와 테마

### 4.1 언어 전환

지원 언어:
- 한국어
- English

특징:
- 선택한 언어는 쿠키에 저장됩니다.
- 일부 텍스트는 관리자가 UI 텍스트 설정에서 별도로 변경할 수 있습니다.

### 4.2 개인 테마

테마는 사용자별로 저장됩니다.

지원 목적:
- 같은 시스템을 여러 사용자가 써도 각자 선호하는 화면 톤을 유지
- 로그인 화면과 앱 화면의 느낌을 최대한 일관되게 유지

## 5. 대시보드에서 할 수 있는 일

대시보드는 사용자의 기본 업무 화면입니다.

주요 기능:
- 내 VM 목록 조회
- VM 전원 제어
- VM 삭제 요청 성격의 Soft Delete
- VM 목록 CSV 내보내기
- VM 생성 요청
- SSH 키 다운로드 및 교체

## 6. 내 VM 보기

대시보드에서는 본인이 접근 권한을 가진 VM만 볼 수 있습니다.

표시 정보 예:
- VM 이름
- 상태
- 그룹
- IP
- CPU / 메모리 / 디스크
- 노드 정보

권한 원칙:
- 일반 사용자는 본인 그룹에 속한 VM만 조회 가능합니다.
- 관리자 계정은 별도 관리자 화면에서 더 많은 정보를 확인합니다.

## 7. VM 전원 제어

가능한 작업:
- 시작
- 중지
- 재부팅

동작 특성:
- 실제 전원 제어는 Proxmox API를 통해 수행됩니다.
- 성공 시 UI 상태와 감사로그에 반영됩니다.

주의:
- 권한 없는 VM에는 작업할 수 없습니다.
- Proxmox 연결 상태에 따라 즉시 반영되지 않을 수 있습니다.

## 8. VM 삭제

사용자 화면의 삭제는 기본적으로 Soft Delete입니다.

의미:
- VM 레코드를 바로 영구 삭제하지 않고 삭제 상태로 전환합니다.
- 운영자가 필요 시 복구나 후속 처리를 할 수 있습니다.

주의:
- Soft Delete와 Proxmox 영구 삭제는 다릅니다.
- 영구 삭제는 관리자 권한이 필요합니다.

## 9. CSV 내보내기

대시보드에서 현재 권한 범위의 VM 목록을 CSV로 내보낼 수 있습니다.

용도:
- 자산 목록 정리
- 운영 현황 공유
- 보고용 정리

## 10. VM 생성 요청

사용자는 직접 VM을 만들지 않고 “요청”을 제출합니다.

요청 입력 항목:
- 그룹
- 인스턴스 타입
- OS
- 수량
- 추가 디스크
- 요청 사유

설계 의도:
- 사용자가 인프라 세부 설정을 직접 맞추지 않게 함
- 공통 표준 이미지를 사용하도록 유도
- 승인과 쿼터 검증을 통해 무분별한 증설을 막음

### 10.1 OS 선택

사용자는 화면에서 지원되는 OS 중 하나를 선택합니다.

현재 운영 기준:
- Ubuntu 24.04
- Rocky Linux 10

선택한 OS는 관리자 승인 화면에서 해당 Cloud Image 자동 선택의 힌트로 사용됩니다.

### 10.2 쿼터와 요청 제한

요청은 그룹 쿼터 정책의 영향을 받습니다.

예:
- VM 수 제한
- CPU 제한
- 메모리 제한
- 디스크 제한

정책 초과 시:
- 요청이 제한되거나
- 관리자 검토가 필요할 수 있습니다.

## 11. SSH 키 다운로드

Linux VM에 SSH 키가 연결된 경우 대시보드에서 키를 다운로드할 수 있습니다.

특징:
- 민감 작업이므로 OTP 재인증이 필요합니다.
- 다운로드는 단순 파일 제공이 아니라 보호된 챌린지 흐름을 거칩니다.

의도:
- 로그인 세션만으로는 민감 파일이 바로 내려가지 않도록 보호

## 12. SSH 키 교체

키 교체는 기존 SSH 키를 새 키로 바꾸는 기능입니다.

동작 순서:
1. 키 교체 버튼 클릭
2. OTP 입력
3. 서버에서 OTP 검증
4. QEMU Guest Agent를 통해 VM 내부 키 교체
5. 새 키 다운로드 안내

중요:
- 이전 키는 즉시 무효화됩니다.
- VM이 실행 중이어야 하며 Guest Agent가 동작해야 합니다.
- Windows VM에는 기본적으로 적용되지 않습니다.

## 13. 자주 묻는 질문

### 13.1 언어를 바꿨는데 일부 문구가 그대로입니다

원인:
- 일부 문구는 템플릿 문자열이고
- 일부 문구는 관리자 UI 텍스트 설정값을 사용합니다.

즉, 한 번에 모두 바뀌지 않을 수 있습니다.

### 13.2 키 다운로드나 키 교체 때 OTP를 다시 묻는 이유는 무엇인가요

SSH 키와 같은 민감 정보는 로그인 이후에도 추가 보호가 필요하기 때문입니다.  
이는 Proxmox Horizon의 기본 보안 철학입니다.

### 13.3 내가 요청한 VM이 바로 생성되지 않습니다

VM 생성은 자동 즉시 실행이 아니라 관리자 승인과 배포 절차를 거칩니다.  
또한 노드, 이미지, 쿼터, 정책 상태에 따라 처리 시간이 달라질 수 있습니다.
